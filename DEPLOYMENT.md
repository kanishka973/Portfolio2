# Deployment Guide

## Quick Deployment

### Option 1: Automated Script (Recommended)
Run the PowerShell script:
```powershell
.\deploy.ps1
```

### Option 2: Manual Steps

#### 1. Push to GitHub

**First time setup:**
```powershell
# Authenticate with GitHub
gh auth login

# Create repository and push
gh repo create Portfolio-Builder --public --source=. --remote=origin --push
```

**Subsequent pushes:**
```powershell
git add .
git commit -m "Your commit message"
git push origin main
```

#### 2. Deploy to Vercel

**First time setup:**
```powershell
# Authenticate with Vercel
vercel login

# Deploy to production
vercel --prod
```

**Subsequent deployments:**
```powershell
vercel --prod
```

## Project Structure

- `api/index.ts` - Vercel serverless function handler
- `vercel.json` - Vercel configuration
- `dist/public/` - Built static files (generated by `npm run build`)

## Build Commands

- `npm run build` - Build the project for production
- `npm run dev` - Run development server
- `npm start` - Start production server

## Environment Variables

No environment variables are required for basic deployment. The project is configured to work out of the box.

## Troubleshooting

### GitHub Authentication Issues
- Make sure GitHub CLI is installed: `winget install --id GitHub.cli`
- Run `gh auth login` and complete browser authentication

### Vercel Deployment Issues
- Make sure Vercel CLI is installed: `npm install -g vercel`
- Run `vercel login` and complete browser authentication
- Check that `dist/public` directory exists (run `npm run build` first)

### Build Errors
- Make sure all dependencies are installed: `npm install`
- Check TypeScript errors: `npm run check`
- Rebuild: `npm run build`
